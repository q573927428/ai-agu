## 1. 环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 用途 | 安装参考 |
|------|---------|------|---------|
| Python | ≥ 3.10 | 运行 FastAPI 后端 | [python.org](https://www.python.org/) |

| Docker & Docker Compose (可选) | 最新 | 容器化一键部署 | [docker.com](https://www.docker.com/products/docker-desktop) |

> **本地 MySQL 必须已安装并启动**，才能进行本地开发模式运行。

```bash
# 检查 MySQL 服务状态（Windows 管理员 PowerShell）
```

```bash
cd backend
pip install -r requirements.txt
```

### 1.4 安装前端依赖

```bash
cd frontend
pnpm install
```

## 2. 方式一：Docker Compose 一键部署

**适合**：快速体验完整功能，无需手动配置本地开发环境。

### 2.1 启动所有服务

在项目根目录（`ai-agu/`）执行：

```bash
docker-compose up -d
```

此命令会依次启动三个容器：

| 容器名 | 镜像 | 端口 | 说明 |
|--------|------|------|------|
| `agu-mysql` | mysql:8.0 | 3306 | MySQL 数据库 |
| `agu-backend` | 本地构建 | 8000 | FastAPI 后端 |
| `agu-frontend` | 本地构建 | 3000 | Nuxt4 前端 |

> ⚠️ **首次启动**会构建后端和前端镜像，耗时约 5-10 分钟，请耐心等待。

### 2.2 查看容器状态

```bash
# 查看所有容器是否正常运行
docker ps

# 查看日志
docker-compose logs -f        # 查看所有服务日志
docker-compose logs -f backend  # 仅查看后端日志
```

### 2.3 初始化数据库

Docker 启动后，MySQL 容器会自动创建数据库 `agu_quant`，但还需要初始化数据表：

```bash
docker exec -it agu-backend python scripts/init_db.py
```

### 2.4 填充演示数据

```bash
docker exec -it agu-backend python scripts/seed_demo_data.py
```

### 2.5 停止服务

```bash
docker-compose down        # 停止并移除容器
docker-compose down -v     # 停止并移除容器+数据卷（会清空数据库）
```

## 4. 方式三：完整真实数据运行（推荐）

**适合**：从 AkShare 拉取真实 A 股数据，运行完整量化流水线。

### 4.1 整体流程

```
[AkShare真实数据源]                    [MySQL数据库]
      │                                     │
      ▼ python scripts/fetch_real_data.py    ▼
  ┌─────────────────────┐           ┌────────────────┐
  │ 拉取全市场股票基础信息  │ ───────►  │ stock_basic    │
  │ 拉取当日日K线行情数据  │ ───────►  │ stock_daily    │
  │ 拉取宏观经济数据      │ ───────►  │ macro_data     │
  └─────────────────────┘           └────────────────┘
                                             │
      ▼ python scripts/run_pipeline.py       ▼
  ┌─────────────────────┐           ┌────────────────┐
  │ ① 因子计算(50个因子) │ ───────►  │ factor_store   │
  │ ② 标签生成          │           │ prediction     │
  │ ③ 预测             │ ───────►  │ ranking_snapshot│
  │ ④ 排名生成          │ ───────►  │                │
  └─────────────────────┘           └────────────────┘
                                             │
      ▼ python scripts/train_model.py        ▼
  ┌─────────────────────┐           ┌────────────────┐
  │ 训练LightGBM模型     │ ───────►  │ model_record    │
  └─────────────────────┘           │ + .txt模型文件   │
                                    └────────────────┘
```

### 4.2 拉取真实数据

```bash
cd backend

# 拉取最近一个交易日的全部真实数据（全市场约5500只A股）
python scripts/fetch_real_data.py

# 拉取指定日期的数据
python scripts/fetch_real_data.py 2026-06-05

# 测试模式：仅拉取前100只股票（快速验证脚本是否正常）
python scripts/fetch_real_data.py 2026-06-05 --top 100

# 拉取最近30个交易日的历史数据（推荐，这样有足够数据训练模型）
python scripts/fetch_real_data.py --history 30

# 跳过宏观数据（如果AkShare接口不稳定时可以临时跳过）
python scripts/fetch_real_data.py --skip-macro
```

**脚本自动执行以下操作**：

| 步骤 | 功能 | 写入的表 | 数据来源（AkShare接口） |
|------|------|---------|------------------------|
| ① | 全市场A股基础信息 | `stock_basic` | `ak.stock_info_a_code_name()` |
| ② | 全市场实时行情数据（日K线） | `stock_daily` | `ak.stock_zh_a_spot()` |
| ③ | 宏观经济数据 | `macro_data` | GDP/CPI/PMI/M2/Shibor/国债收益率/北向资金 |

> **说明**：`stock_daily` 表存储 **日K线数据**（open/high/low/close/volume/amount），包含完整OHLC。不存储分钟级K线。

### 4.3 运行核心流水线

拉取到数据后，运行量化流水线进行因子计算和预测：

```bash
# 运行完整流水线（因子计算 → 标签生成 → 预测 → 排名）
python scripts/run_pipeline.py 2026-06-05

# 快速测试模式（仅处理前50只股票）
python scripts/run_pipeline.py 2026-06-05 50
```

> **注意**：首次运行时 `Predictor` 会提示"没有活跃模型"，跳过预测和排名步骤，这是正常的——需要先训练模型。

### 4.4 训练模型

积累至少 **3 个交易日** 的因子数据后训练模型：

```bash
python scripts/train_model.py
```

> ⚠️ **建议先用 `--history 30` 拉取 30 天历史数据**，这样数据量足够模型训练出有意义的结果。

### 4.5 生产模式每日运行

系统部署上线后，每天只需两步：

```bash
# 第一步：拉取当日数据
cd backend
python scripts/fetch_real_data.py

# 第二步：运行量化流水线
python scripts/run_pipeline.py
```

## 5. 核心脚本说明

项目 `backend/scripts/` 目录下提供多个辅助脚本，按执行顺序排序如下：

### 5.1 `init_db.py` — 数据库初始化

```bash
cd backend
python scripts/init_db.py
```

| 功能 | 说明 |
|------|------|
| 创建数据库 | `agu_quant`（如果不存在） |
| 创建用户 | `agu_user`（如果不存在） |
| 创建数据表 | 所有 ORM 模型对应的表（stock_basic, stock_daily, factor_store, prediction, ranking_snapshot 等） |

> **前置条件**：MySQL 服务已启动，root 密码正确（默认 `root123`）。

### 5.2 `fetch_real_data.py` — 从AkShare拉取真实A股数据

```bash
cd backend

# 基本用法
python scripts/fetch_real_data.py

# 拉取指定日期
python scripts/fetch_real_data.py 2026-06-05

# 拉取历史数据
python scripts/fetch_real_data.py --history 30

# 限制数量（测试用）
python scripts/fetch_real_data.py --top 100
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `date` | 最近交易日 | 要拉取的日期 (YYYY-MM-DD) |
| `--history N` | 0 | 拉取最近 N 个交易日的历史数据 |
| `--top N` | 0 (全部) | 仅处理前 N 只股票 |
| `--skip-macro` | false | 跳过拉取宏观经济数据 |
| `--delay N` | 0.1 | AkShare 请求间隔秒数 |

**数据写入内容**：

| 表名 | 数据说明 | 写入策略 |
|------|---------|---------|
| `stock_basic` | 全市场A股基础信息（代码/名称） | 按 `stock_code` 主键去重，已存在则跳过 |
| `stock_daily` | 日K线数据（O/H/L/C/Vol/金额/PE/PB/市值） | 按 `(stock_code, trade_date)` 唯一约束，已存在则跳过 |
| `macro_data` | 宏观经济数据（GDP/CPI/PMI/M2/Shibor/国债收益率/北向资金） | 按 `data_date` 唯一约束，已存在则跳过 |

### 5.4 `run_pipeline.py` — 一键运行完整量化流水线

```bash
cd backend

python scripts/fetch_real_data.py --history 3 --top 10 --workers 3

# 使用当天日期运行
python scripts/run_pipeline.py

# 指定日期运行
python scripts/run_pipeline.py 2026-06-05

# 仅处理前 10 只股票（快速验证模式）
python scripts/run_pipeline.py 2026-06-05 10
```

**流水线四大步骤**：

```
[1/4] 因子计算    → FactorEngine.compute_all()    → 写入 factor_store 表（50个因子）
[2/4] 标签生成    → LabelGenerator.generate_labels() → 生成未来20日收益率标签
[3/4] 预测        → Predictor.predict_daily()      → 写入 prediction 表（需模型已训练）
[4/4] 排名生成    → RankingService.save_ranking_snapshot() → 写入 ranking_snapshot 表（TOP50）
```

### 5.5 `train_model.py` — 训练LightGBM模型

```bash
cd backend
python scripts/train_model.py
```

**执行流程**：

```
① 从 stock_daily 获取所有交易日
② 对每个交易日调用 FactorEngine.compute_all() 计算因子
③ 因子数据保存到 factor_store 表
④ 检查因子数据是否 ≥ 3 天（不足则报错退出）
⑤ 调用 Trainer.train() 训练 LightGBM 回归模型
⑥ 保存模型文件 (.txt) 到 models/ 目录
⑦ 记录训练信息到 model_record 表，设为活跃模型
```

**模型训练结果输出示例**：

```bash
训练范围: 2026-04-25 ~ 2026-05-28
训练数据: 150000 条, 验证数据: 65000 条, 特征: 45 个
✅ 训练完成: 版本=v20260605_143000, IC=0.052, RankIC=0.048
```

## 7. 常见问题

### Q1: MySQL 连接失败

**错误信息**：
```
sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) (2003, Can't connect to MySQL server on 'localhost:3306')
```

**解决方法**：
```bash
# 1. 确认 MySQL 服务是否启动
net start | findstr mysql

# 2. 如果未启动，启动服务
net start MySQL97

# 3. 检查端口是否被占用
netstat -ano | findstr :3306
```

### Q2: Docker 容器启动失败

```bash
# 查看容器日志定位问题
docker logs agu-mysql     # 查看 MySQL 日志
docker logs agu-backend   # 查看后端日志
docker logs agu-frontend  # 查看前端日志
```

### Q3: 端口被占用

```bash
# 查看端口占用情况
netstat -ano | findstr :8000
netstat -ano | findstr :3000
netstat -ano | findstr :3306

# 修改端口：编辑 docker-compose.yml 或 .env 文件修改对应的端口映射
```

### Q4: 依赖安装失败

```bash
# 后端依赖安装失败时
cd backend
pip install --upgrade pip
pip install -r requirements.txt

# 前端依赖安装失败时
cd frontend
pnpm store prune        # 清理缓存
pnpm install            # 重新安装
```

### Q5: 数据库初始化报错

**错误信息**：`Error 1396 (HY000): Operation CREATE USER failed`

**原因**：用户已存在。解决方案：跳过用户创建步骤，直接初始化表：

```python
# 手动修改 init_db.py，注释掉用户创建逻辑
# conn.execute(text(f"CREATE USER IF NOT EXISTS..."))
# conn.execute(text(f"GRANT ALL PRIVILEGES..."))
```

### Q6: 前端访问后端接口返回 502/网络错误

**检查顺序**：
1. 后端是否正常运行：访问 `http://localhost:8000/api/v1/health`
2. 前端 `.env` 中 `NUXT_PUBLIC_API_BASE` 是否配置正确（默认 `http://localhost:8000`）
3. Docker 环境下，检查容器网络是否互通：`docker exec agu-frontend curl http://agu-backend:8000`

### Q7: 训练模型时提示"因子数据不足"

**原因**：`factor_store` 表中不足 3 个交易日的数据。

**解决方法**：
```bash
# 先用真实数据拉取多日历史
python scripts/fetch_real_data.py --history 30

# 再运行训练
python scripts/train_model.py
```

### Q8: AkShare 接口报错

**错误信息**：
```
akshare 请求超时 / 接口返回空数据 / 连接错误
```

**原因**：AkShare 依赖第三方金融数据网站，可能因网络或频率限制导致失败。

**解决方法**：
```bash
# 增加请求间隔，降低频率
python scripts/fetch_real_data.py --delay 1.0

# 跳过不稳定的接口
python scripts/fetch_real_data.py --skip-macro

# 如果非交易时段，ak.stock_zh_a_spot() 会返回空数据，这是正常的
```
