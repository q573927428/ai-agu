## 2. 快速启动（推荐）

完整流程：拉取真实A股数据 → 量化分析 → 训练模型 → 启动服务。

### 2.1 启动 MySQL

```bash
net start MySQL97      # Windows 启动 MySQL 服务
```

### 2.2 初始化数据库

```bash
cd backend
python scripts/init_db.py
```

自动创建数据库 `agu_quant`、用户 `agu_user` 及所有数据表。

### 2.3 拉取真实数据

股票基础信息只需拉取一次即可获取所有A股，K线数据建议拉取多日用于训练模型。

```bash
# ① 首次：拉取全市场股票基础信息 + 当日K线（约5500只A股）
python scripts/fetch_real_data.py

# ② 推荐：拉取最近 60 天历史K线数据（覆盖全市场，数据量充足训练效果更好）
python scripts/fetch_real_data.py --history 60

# ③ 也可指定日期拉取
python scripts/fetch_real_data.py 2026-06-05

# ④ 测试模式：仅处理前 100 只股票的K线数据（快速验证脚本是否正常）
python scripts/fetch_real_data.py --top 100

# ⑤ 断点续传模式，跳过利润表、资产负债表、现金流量表，__只拉 fina_indicator（财务指标）__
python scripts/fetch_financial_full.py --resume --skip-tables income balancesheet cashflow

# 只拉取 2026-06-05 的行情 / 基础数据，跳过宏观数据、跳过财务数据。
cd backend ; python scripts/fetch_real_data.py 2026-06-05 --skip-macro --skip-financial

```

> **说明**：`--top N` 仅限制K线数据处理数量，股票基础信息始终拉取全部。日常使用无需加 `--top`。全市场K线按时间批量拉取速度很快，建议设置较大历史天数确保模型训练数据充足。

### 2.4 运行量化流水线

```bash
python scripts/run_pipeline.py

python scripts/run_pipeline.py 2026-06-05
```

依次执行：因子计算(50个) → 标签生成 → 预测 → 排名生成。首次运行因无模型会跳过预测步骤，属正常现象。

### 2.5 训练模型

积累 **3 天以上** 因子数据后训练：

```bash
python scripts/train_model.py
```

### 2.6 启动服务

```bash
# 终端1 - 后端
cd backend; python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 终端2 - 前端
cd frontend; pnpm dev
```

### 生产模式每日运行

部署上线后，每天只需两步：

```bash
python scripts/fetch_real_data.py   # 拉取当日数据
python scripts/run_pipeline.py      # 运行流水线
```

## 3. Docker 部署

适用于快速体验，无需手动配置环境。

```bash
# 一键启动所有服务
docker-compose up -d

# 初始化数据库
docker exec -it agu-backend python scripts/init_db.py

# 填充演示数据（无需 AkShare）
docker exec -it agu-backend python scripts/seed_demo_data.py
```

访问地址：前端 `http://localhost:3000`，后端 `http://localhost:8000`

## 5. 常用脚本速查

| 脚本 | 用途 | 示例 |
|------|------|------|
| `init_db.py` | 初始化数据库和表 | `python scripts/init_db.py` |
| `fetch_real_data.py` | 拉取真实A股数据 | `python scripts/fetch_real_data.py --history 60` |
| `seed_demo_data.py` | 填充演示数据 | `python scripts/seed_demo_data.py` |
| `run_pipeline.py` | 运行量化流水线 | `python scripts/run_pipeline.py 2026-06-05` |
| `train_model.py` | 训练LightGBM模型 | `python scripts/train_model.py` |
| `check_db.py` | 查看数据库概况 | `python scripts/check_db.py` |

### fetch_real_data.py 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `date` | 最近交易日 | 指定日期 (YYYY-MM-DD) |
| `--history N` | 0 | 拉取最近 N 天历史数据 |
| `--top N` | 全部 | 仅处理前 N 只股票 |
| `--skip-macro` | false | 跳过宏观经济数据 |
| `--skip-financial` | false | 跳过财务数据 |
| `--delay N` | 0.1 | 请求间隔秒数 |

### 数据清空

```bash
mysql -u agu_user -pagu123 -h localhost -P 3306 agu_quant -e "SET FOREIGN_KEY_CHECKS = 0; TRUNCATE TABLE stock_daily; TRUNCATE TABLE income; TRUNCATE TABLE balancesheet; TRUNCATE TABLE cashflow; TRUNCATE TABLE fina_indicator; TRUNCATE TABLE macro_data; TRUNCATE TABLE factor_store; TRUNCATE TABLE prediction; TRUNCATE TABLE ranking_snapshot; TRUNCATE TABLE model_record; TRUNCATE TABLE stock_basic; SET FOREIGN_KEY_CHECKS = 1;"
```

## 7. 常见问题

### MySQL 连接失败
```bash
net start | findstr mysql    # 检查 MySQL 是否启动
netstat -ano | findstr :3306 # 检查端口
```

### 端口被占用
```bash
netstat -ano | findstr :8000  # 查看占用端口的进程
# 修改 docker-compose.yml 或启动命令中的端口映射
```

### 因子数据不足（训练模型时）
```bash
# 拉取多日历史数据后再训练
python scripts/fetch_real_data.py --history 60
python scripts/train_model.py
```

### AkShare 接口报错
```bash
# 增加请求间隔，降低频率
python scripts/fetch_real_data.py --delay 1.0

# 跳过不稳定接口
python scripts/fetch_real_data.py --skip-macro
```
